Realizado por Juan Camilo Dousdebes Morales y Julian David Mancipe Ruiz ADSO Abril del 2025

Variables implementadas

Información General

  • KLOC las kilolineas estimadas del proyecto
  • wage el sueldo de los programadores
  • personel los miembros del proyecto
  • idealTime los meses que idealmente va a tomar le desarrollo
  • mode el modo de desarrollo, dependiendo del modo se cambia el valor de los ajustes A, B, C y D

Atributos del producto

  • RELY Fiabilidad requerida del software
  • DATA Tamaño de la base de datos
  • CPLX Complejidad del producto

Atributos de la computadora

  • TIME Restricciones del tiempo de ejecución
  • STOR Restricciones del almacenamiento principal
  • VIRT Inestabilidad de la máquina virtual
  • TURN Tiempo de respuesta del computador

Atributos del personal

  • ACAP Capacidad del analista
  • AEXP Experiencia en la aplicación
  • PCAP Capacidad de los programadores
  • VEXP Experiencia en el sistema operativo utilizado
  • LEXP Experiencia en el lenguaje de programación

Atributos del proyecto

  • MODP Uso de prácticas de programación modernas
  • TOOL Uso de herramientas software
  • SCED Restricciones en la duración del proyecto

Ejemplo de calculo e interpretación

Se desea calcular un proyecto simple para una pizzería la cual registra compras en una base de datos y se puede acceder a esta mediante una API.

KLOC = 5.5 Se estimó con otro modelo que el proyecto va a tomar 5500 lineas de codigo wage = $1.423.500 Se les va a pagar el mínimo a los empleados personel = ? No se especifica la cantidad de empleados en este caso, por lo que es una incognita a resolver idealTime = 6 Se espera que el proyecto esté terminado a mediados de año mode = Semilibre Se define que el modo de desarrollo va a ser Semilibre, por lo que A = 3 El ajuste A pasa a ser 3 B = 1,12 El ajuste B pasa a ser 1,12 C = 2,5 El ajuste C pasa a ser 2,5 D = 0,35 El ajuste D pasa a ser 0,35 RELY = alta = 1,15 Se define que el proyecto sea altamente fiable, por lo que es un valor de ajuste de 1,15 DATA = nominal = 1 Se define que el tamaño de la base de datos es nominal, por lo que el valor de ajuste pasa a ser 1 CPLX = bajo = 0,85 Se define que la complejidad del producto es baja, por lo que el valor de ajuste es 0,85 TIME = nominal = 1 Se
