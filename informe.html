Informe de la calculadora COCOMO81

Realizado por Juan Camilo Dousdebes Morales y Julian David Mancipe Ruiz ADSO Abril del 2025

Variables implementadas

Información General

  • KLOC las kilolineas estimadas del proyecto
  • wage el sueldo de los programadores
  • P los miembros del proyecto
  • Tdev los meses que idealmente va a tomar el desarrollo en meses
  • mode el modo de desarrollo, dependiendo del modo se cambia el valor de los ajustes A, B, C y D

Atributos del producto

  • RELY Fiabilidad requerida del software
  • DATA Tamaño de la base de datos
  • CPLX Complejidad del producto

Atributos de la computadora

  • TIME Restricciones del tiempo de ejecución
  • STOR Restricciones del almacenamiento principal
  • VIRT Inestabilidad de la máquina virtual
  • TURN Tiempo de respuesta del computador

Atributos del personal

  • ACAP Capacidad del analista
  • AEXP Experiencia en la aplicación
  • PCAP Capacidad de los programadores
  • VEXP Experiencia en el sistema operativo utilizado
  • LEXP Experiencia en el lenguaje de programación

Atributos del proyecto

  • MODP Uso de prácticas de programación modernas
  • TOOL Uso de herramientas software
  • SCED Restricciones en la duración del proyecto

Variables calculadas

  • E La quantificación del esfuerzo puesto en el proyecto
  • m(X) La quantificación de todos los factores que puedan afectar al proyecto
  • totalCost El costo total del proyecto

Ejemplo de calculo e interpretación

Se desea calcular un proyecto simple para una pizzería la cual registra compras en una base de datos y se puede acceder a esta mediante una API.

KLOC = 5 Se estimó con otro modelo que el proyecto va a tomar 5500 lineas de codigo wage = $2.000.000 Se les va a pagar dos millones a cada desarrollador por mes personel = ? No se especifica la cantidad de empleados en este caso, por lo que es una incognita a resolver idealTime = 6 Se espera que el proyecto esté terminado en 6 meses mode = Semilibre Se define que el modo de desarrollo va a ser Semilibre, por lo que A = 3 El ajuste A pasa a ser 3 B = 1,12 El ajuste B pasa a ser 1,12 C = 2,5 El ajuste C pasa a ser 2,5 D = 0,35 El ajuste D pasa a ser 0,35 RELY = alto = 1,15 Se define que el proyecto sea altamente fiable, por lo que es un valor de ajuste de 1,15 DATA = nominal = 1 Se define que el tamaño de la base de datos es nominal, por lo que el valor de ajuste pasa a ser 1 CPLX = bajo = 0,85 Se define que la complejidad del producto es baja, por lo que el valor de ajuste es 0,85 TIME = nominal = 1 Se define que no hay restricciones respecto al tiempo de ejecución, por lo que el valor de ajuste es 1 STOR = alto = 1,06 Se define que hay unas restricciónes de almacenamiento altas dado a los servicios elegidos, por lo que se le da un valor de ajuste de 1,06 VIRT = bajo = 0,87 Se define que hay baja instabilidad en la maquina virtual, dado a la simpleza del sistema, por lo que se le da un valor de ajuste de 0,87 TURN = muy alto = 1,15 Se define que el tiempo de respuesta debe ser muy alto, por lo que el valor ajustado pasa a ser 1,15 ACAP = nominal = 1 Se define que la capacidad del analista es normal, por lo que el valor de ajuste es 1 AEXP = alto = 0,91 Se define que la experiencia de los desarrolladores debe ser alta, por lo que el valor de ajuste es 0,91 PCAP = alto = 0,87 Se define que los desarrolladores deben tener altas capacidades, por lo que el valor de ajuste es 0,87 VEXP = alto = 0,9 Se define que los desarrolladores deben estar altamente familiarizados con el sistema operativo, por lo que el valor de ajuste es 0,9 LEXP = alto = 0,95 Se define que los desarrolladores deben estar altamente familiarizados con el lenguaje de programación, por lo que el valor de ajuste es 0,95 MODP = nominal = 1 Se define que el software utilice practicas medianamente modernas, por lo que el valor de ajuste es 1 TOOL = muy alto = 0,83 Se define que los desarrolladores deben estar muy familiarizados con las herramientas de desarrollo, por lo que el valor de ajuste es 0,83 SCED = alto = 1,04Se define que hay restricciones altas sobre el tiempo de desarrollo el proyecto, por lo que el valor de ajuste es 1,04

Ahora con los 15 valores de ajuste definidos, se calcula el factor final multiplicando a todos los anteriores para obtener un ajuste según las condicines del proyecto

m(X) = RELY * DATA * CPLX * TIME * STOR * VIRT * TURN * ACAP * AEXP * PCAP * VEXP * LEXP * MODP * TOOL * SCED

Que en el caso del ejemplo equivale a

m(X) = 1,15 * 1 * 0,85 * 1 * 1,06 * 0,87 * 1,15 * 1 * 0,91 * 0,87 * 0,9 * 0,95 * 1 * 0,83 * 0,87 * 0,9 * 1 * 0,83 * 1,04 m(X) = ~0,4

Posteriormente se calcula el esfuerzo total del proyecto con los ajustes A y B multiplicando la cantidad de kilolineas y luego se le aplica el ajuste final que toma en cuenta todas las condiciones del proyect

E = A * (KLOC)B * m(X) E = ~7,28

No se calcula el tiempo ideal de desarrollo (Tdev) en este caso ya que fue especificado previamente, pero de hacerlo si no fuese el caso entonces se calcularía con la formula

Tdev = c * (E)D

Ya que en este caso se desconoce cual es la cantidad de desarrolladores necesaria, entonces se divide el esfuerzo por el tiempo ideal de desarrollo en meses

P = E/Tdev P = 1.21 que redondeado daría 2 personas

Finalmente se calcula el costo total del desarrollo multiplicando el salario mensual de cada empleado por los meses de desarrollo

totalCost = (P * wage) * Tdev totalCost = (2 * 2.000.000) * 6 = $24.000.000

Siguiendo el modelo COCOMO81, el proyecto va a necesitar dos personas para desarollarlo en 6 meses y va a costar $24.000.000
